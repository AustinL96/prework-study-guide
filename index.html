Prework Study Guide

HTML

  • The head element contains information about the webpage
  • The head element's information is not visible when a user looks at a webpage.
  • The body element represents the visible content shown to the user
  • The head element's information uses metadata to help the browser render the page correctly. This includes data like the character set, description, keywords, author, and viewports.
  • The charset element sets our character encoding.
  • The http-equiv element sets an HTTP header for content.
  • The viewport element sets the default view for our webpage.
  • The "p" element defines a paragraph.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS is for applying an element directly in the HTML file. They're useful for quick and permanent changes but less flexible.
  • Internal CSS style sheet contains CSS rules for the webpage in the head section of the HTML file. The rules apply only to that page, and a single change will apply to all tagged page elements.
  • External CSS style sheet is the most common. This method stores all CSS rules in a separate file, allowing for easier changes.
  • The * element applies CSS rules to all visible elements on a page.
  • A comma can allow multiple selectors (such as a header & footer) to be affected by a CSS rule. This cuts down on lines of repeated code.
  • The block element takes up the entire width on its line.
  • The link element creates a connection to an external file.
  • The rel attribute specifies the relation between the current document and the linked document resource.
  • The href attribute specifies the location (URL) of the external source, using a relative path. Meaning it is derived from the current location.
  • In HTML, a section element may have a class attribute. To target elements with a specific class, write a period (.) followed by the name of the class.
