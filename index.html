HTML

  • The head element contains information about the webpage
  • The head element's information is not visible when a user looks at a webpage.
  • The body element represents the visible content shown to the user
  • The head element's information uses metadata to help the browser render the page correctly. This includes data like the character set, description, keywords, author, and viewports.
  • The charset element sets our character encoding.
  • The http-equiv element sets an HTTP header for content.
  • The viewport element sets the default view for our webpage.
  • The "p" element defines a paragraph.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS is for applying an element directly in the HTML file. They're useful for quick and permanent changes but less flexible.
  • Internal CSS style sheet contains CSS rules for the webpage in the head section of the HTML file. The rules apply only to that page, and a single change will apply to all tagged page elements.
  • External CSS style sheet is the most common. This method stores all CSS rules in a separate file, allowing for easier changes.
  • The * element applies CSS rules to all visible elements on a page.
  • A comma can allow multiple selectors (such as a header & footer) to be affected by a CSS rule. This cuts down on lines of repeated code.
  • The block element takes up the entire width on its line.
  • The link element creates a connection to an external file.
  • The rel attribute specifies the relation between the current document and the linked document resource.
  • The href attribute specifies the location (URL) of the external source, using a relative path. Meaning it is derived from the current location.
  • In HTML, a section element may have a class attribute. To target elements with a specific class, write a period (.) followed by the name of the class.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout main: switches to the main branch if your are not in it
  • pwd: displays the current working directory, and stands for "print working directory"
  • ls: displays a list of files and directories
  • cd "name": changes directory
  • cd ..: navigates your directory backwards
  • mkdir "name": makes a new directory
  • touch "name": creates a new file inside a directory
  • start "name": starts a file

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Javascript is read from top to bottom, meaning the order in which we set up code matters.
  • The "console.log" method outputs a message to the web console by adding an argument.
  • An "if" statement is a way of introducing decision-making into our code, but only if the statement is true or meets a condition. If it is, it will do whatever it is asked between "{ }" brackets.
  • An "else if" statement can be used to provide more than one condition to an "if" statement.
  • The "0" and "null" values are falsey, meaning a computer won't execute code related to them.
  • The strict equality operator "===" checks to see if two values are equal, returning a Boolean result "true" if the values are.
  • An array is a single variable that is used to hold a group of data, usually data that is related in some way.
  • An index is a unique number in an array tied to individual data. Arrays are zero-indexed and sequential, meaning the first item in any array is 0, not 1.
  • A "for" loop can perform a task on all items in an array by executing a code block over and over, until a specified condition evaluates to false.
  • Functions set instructions for a computer to perform a task, but unlike loops or other statements, they do not automatically execute when JavaScript is run.
  • "Math.floor()" and "Math.random()" are both peroperties of JavaScript's built-in "Math" object that can be used together to generate a random number.
